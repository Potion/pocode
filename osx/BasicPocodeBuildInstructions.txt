Pocode build:
	1) build OSX (any style) -- moves some headers
	2) ios simulator target
	3) Product - build for profiling (makes a release config build)
	4) ios device target
	5) Product - build for profiling (makes a release config build)
	rinse and repeat steps 3 - 7, but use regular (debug build)

This should result in the following built libraries in /pocode/libraries/pocode

├── libpocode_iOS.a
├── libpocode_iOS_d.a
├── libpocode_iOS_simulator.a
├── libpocode_iOS_simulator_d.a

Where _d denotes a debug build
