Module Entry Points

Now that we have a node implementation and an exporter which exports it to OpenNI, all that is left is to let OpenNI know which exporters exist in your module.

The best way to do so is to create a new cpp file, and in it:

  • Include OpenNI C++ registration header: 
        #include <XnModuleCppRegistratration.h>
  • Declare this module. This is done by the following line: 
        XN_EXPORT_MODULE(xn::Module)
    If you want your module to behave differently than default (for example, do some extra initialization upon shared library loading), you can create a new class, inheriting from xn::Module, and export it instead.
  • Declare all exporters using the corresponding macros. For example, to declare a HandsGenerator exporter: 
        XN_EXPORT_HANDS(xn::MyHandsExporter)
